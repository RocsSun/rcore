# Rclib

rcore lib，这里和原本教程中的不一致，这里改成了rclib，原来是user_lib

此crate是U模式下的内容，所有权限都在u模式。如果需要更高的权限，则需要调用ABI（即syscall）。有os中的代码来实现。


- rclib/src/bin/*.rs ：各个应用程序
- rclib/src/*.rs ：用户库（包括入口函数、初始化函数、I/O 函数和系统调用接口等）
- rclib/src/linker.ld ：应用程序的内存布局说明。

bin可以理解为平常开发的软件状态
src可以理解为平常开发是遇到的系统接口调用。

其实这一层所有的代码都是在u模式。包括abi接口提供的api。只不过他内部实现了`ecall`这种trap的汇编指令，让错做系统来处理。在平常软件开发中，根本就碰不了s模式的任何东西。

`syscall`函数实现`ecall`的汇编，其ABI规范如下：

- a0(x10)：调用时是调用者传递参数寄存器，eret时，保存返回值，因此他比较特殊，即是传参寄存器，优势返回寄存器。
- a1~a6(X11~X16): 调用这传递参数
- a7(x17): syscall ID，这个在linux中是已经规定好了，
